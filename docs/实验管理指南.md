# YOLO实验管理指南

## 1. 实验结果分类存储

### 1.1 推荐的存储结构

```
runs/
├── train/                          # 训练结果
│   ├── visdrone_yolo11s/           # VisDrone数据集 + YOLO11s模型
│   │   ├── args.yaml              # 训练参数
│   │   ├── results.csv            # 训练指标历史
│   │   ├── weights/               # 模型权重
│   │   │   ├── best.pt           # 最佳模型
│   │   │   └── last.pt           # 最终模型
│   │   └── plots/                 # 训练图表
│   ├── visdrone_yolo11m/           # VisDrone + YOLO11m
│   ├── visdrone_mra_std/           # VisDrone + 改进模型
│   ├── coco_yolo11s/               # COCO + YOLO11s
│   └── dota_yolo11s/               # DOTA + YOLO11s
├── val/                            # 验证结果
│   ├── visdrone_yolo11s_val/
│   └── visdrone_yolo11m_val/
└── detect/                         # 检测结果
    ├── visdrone_yolo11s_detect/
    └── test_images_detect/
```

### 1.2 命名规范

**格式**: `{数据集名称}_{模型名称}_{版本号(可选)}`

**示例**:
- `visdrone_yolo11s` - VisDrone数据集使用YOLO11s模型
- `visdrone_mra_std` - VisDrone数据集使用改进的YOLO11模型
- `coco_yolo11m_v2` - COCO数据集使用YOLO11m模型第2版
- `dota_custom_model` - DOTA数据集使用自定义模型

### 1.3 修改name参数的方法

#### 方法1：命令行参数
```bash
# 训练VisDrone数据集的YOLO11模型
python scripts/train.py --name visdrone_yolo11s --model models/yolov11.yaml

# 训练VisDrone数据集的改进模型
python scripts/train.py --name visdrone_mra_std --model "models/MRA-STD YOLO.yaml"

# 训练UAVDT数据集
python scripts/train.py --name uavdt_yolo11 --data configs/uavdt.yaml --model models/yolov11.yaml

# 使用预训练权重
python scripts/train.py --name visdrone_yolo11s_pretrained --model weights/yolo11s-visdrone.pt
```

#### 方法2：脚本内修改
```python
# 在train.py的main()函数中修改默认值
parser.add_argument('--name', type=str, 
                    default='visdrone_yolo11s_v1',  # 修改这里
                    help='实验名称')
```

#### 方法3：自动生成名称（推荐）
```python
def generate_experiment_name(data_config, model_config):
    """根据数据集和模型自动生成实验名称"""
    # 提取数据集名称
    dataset_name = Path(data_config).stem  # configs/visdrone.yaml -> visdrone
    
    # 提取模型名称
    if model_config.endswith('.pt'):
        model_name = Path(model_config).stem  # weights/yolo11s-visdrone.pt -> yolo11s-visdrone
    else:
        model_name = Path(model_config).stem  # models/yolov11.yaml -> yolov11
    
    return f"{dataset_name}_{model_name}"
```

## 2. Train vs Val 脚本的区别

### 2.1 功能对比

| 功能 | Train脚本中的验证 | 独立Val脚本 |
|------|------------------|-------------|
| **运行时机** | 训练过程中自动执行 | 训练完成后手动执行 |
| **数据集选择** | 固定使用train配置中的val分割 | 可指定任意分割(val/test) |
| **参数灵活性** | 受训练参数限制 | 独立的验证参数 |
| **输出详细程度** | 基础训练监控指标 | 详细的每类别分析 |
| **保存格式** | 训练日志的一部分 | 独立的验证报告 |
| **置信度阈值** | 固定 | 可调整测试不同阈值 |

### 2.2 建议使用策略

1. **保留Train脚本的验证**: 用于训练过程监控和早停
2. **保留独立Val脚本**: 用于最终模型评估和详细分析
3. **快速训练时**: 可以使用`--disable-val`参数关闭训练时验证

### 2.3 使用场景

**Train脚本验证适用于**:
- 监控训练进度
- 实现早停策略
- 快速查看模型性能趋势

**独立Val脚本适用于**:
- 最终模型性能评估
- 不同置信度阈值测试
- 测试集评估
- 详细的每类别分析
- 生成评估报告

## 3. 脚本参数对照表

### 3.1 模型权重参数统一说明

所有脚本现在都支持两种参数名称（互为别名）：

| 脚本 | 主参数名 | 别名参数 | 说明 |
|------|----------|----------|------|
| `train.py` | `--model` | 无 | 支持.yaml和.pt文件 |
| `val.py` | `--model` | `--weights` | 仅支持.pt文件 |
| `detect.py` | `--model` | `--weights` | 仅支持.pt文件 |

### 3.2 推荐用法

```bash
# 训练：使用--model
python scripts/train.py --name experiment --model models/yolov11.yaml

# 验证：两种方式都可以（推荐--model）
python scripts/val.py --name experiment_val --model runs/train/experiment/weights/best.pt
python scripts/val.py --name experiment_val --weights runs/train/experiment/weights/best.pt

# 检测：两种方式都可以（推荐--model）
python scripts/detect.py --name experiment_detect --model runs/train/experiment/weights/best.pt
python scripts/detect.py --name experiment_detect --weights runs/train/experiment/weights/best.pt
```
